The Yankee Candle Index and other creative metrics, Sep 26, 2023

I've been fascinated lately with metrics that, for lack of a better word, are silly. One such instance that comes to mind is the "Waffle House Index", which posits that the FEMA response to a disaster is inversely proportional to the number of Waffle Houses that remain open. If many Waffle Houses are still open, then the disaster wasn't so bad. If your local Waffle House is closed, better hunker down. Ill explore this phenomenom at a later time. One similar metric that I am currently exploring is the Yankee Candle index, which is another tongue-in-cheek metric by sharp-eyed Yelp users, who noticed an uptick in Yankee Candle reviews complaining about a lack of scent in their candles during high rates of COVID transmission. The possible implication: high rates of COVID cause a lack of smell, which causes people to leave bad reviews on the Yankee candle site. To explore this, we need two datasets. First, and probably easiest to obtain, is COVID data. The second, and slightly more intensive, is Yankee Candle review data. Yelp has an API we can hit for this. After that point, I plan on using a regex approach to search comments and perform some descriptive analytics. More to come

Bazaar-Style Languages, and why they can sometimes suck Sep 27, 2023

When I was first getting into programming, I had read a quite interesting article on Python, comparing it to the mathemetical tour-de-force, Wolfram Alpha. The author described Wolfram Alpha as a skyscraper - very, very tall, but isolated language whose success solely relied on the ingenuity and mathematical prowess of its founder. When the founder died, so too would the language (or that was the argument, at least). Python, by contrast, is a bazaar-style language. There is very little in the way of a universal source of truth - rather, there is a thriving community which is continually adding functionality to the language via independently-released modules and packages. Python's success, therefore, depends largely on the community who uses it rather than one specific person.

This article came to me today as I continue my work on the Yankee Candle Index exploratory project. I have decided to switch from the Yelp API to Amazon reviews, largely because I believe they will be more comprehensive. In doing so, I was examining the ways to get review data from Amazon without signing up for developer/associate access. One (creative) way of doing so is by simply webscraping review data from the HTML/CSS instead of using the API. Doing so involves a package called selectorlib, which lets you create a YAML file of a given webpage for use in webscraping. Doing so involves a few slightly unorthodox modules, including lmxl, which itself requires dependencies lmxlib and others. It turns out, these are not Python modules but standard libraries, complicating the install process, espeically on MacOS (Windows has some binary distros). At any rate, I believe this to be one of the few drawbacks of this bazaar-style system. A non-unified development front means that sometimes inconsistent package requisites. A small price to pay, I suppose, for some really cool community contributions.
